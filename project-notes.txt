Для чого потрібен Git?

Git — це система контролю версій, яка допомагає відстежувати зміни в коді та файлах проєкту.

За допомогою Git можна:

 - зберігати історію всіх змін у проєкті;
 - повертатися до попередніх версій файлів;
 - працювати над різними функціями в окремих гілках (branches);
 - об'єднувати зміни від кількох розробників;
 - уникати втрати коду через помилки або випадкове видалення.

Приклад: якщо після внесення змін програма перестала працювати, Git дозволяє повернутися до попередньої робочої версії.

Для чого потрібен GitHub?

GitHub — це веб-платформа для зберігання Git-репозиторіїв у хмарі та спільної роботи над проєктами.

За допомогою GitHub можна:

 - зберігати резервну копію проєкту в Інтернеті;
 - працювати над проєктом разом з іншими розробниками;
 - ділитися своїм кодом з роботодавцями або викладачами;
 - керувати завданнями та обговорювати зміни;
 - автоматизувати тестування та розгортання застосунків.

Приклад: ви створили проєкт на своєму комп'ютері за допомогою Git і завантажили його на GitHub, щоб мати доступ до н
ього з будь-якого пристрою та працювати над ним у команді.
